# Order Flow Screens

## Covered screens
- Customer booking
- Customer order detail
- Staff order handling
- Admin assignment

## Target
All order flows must use the same compact card, status chip, timeline, and bottom action bar language.

## Rules
- Primary action: green filled button
- Secondary action: outlined button with slate/green text depending context
- Selected card: soft green surface and green border/icon
- Waiting/pending state: amber chip
- Active pickup progress: blue chip
- Completed/positive state: green chip
- Utility/support state: purple chip
- Neutral/cancelled state: slate chip or slate text on gray surface
- Timeline: compact vertical timeline
- Keep Riverpod state and order status lifecycle unchanged
